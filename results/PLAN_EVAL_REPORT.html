Top Gaps and Why They Matter

These are the highest-impact blockers from the audit, ranked by severity and delivery risk.

PRD-010 • Critical
Backend truth and cache/reset safety
Without this, data can diverge between server and clients and test resets become unreliable.
PRD-012 • Critical
Per-field timestamp conflict model
Merge logic cannot preserve user edits safely under sync and catalog refreshes.
PRD-016 • Critical
Upgrade continuity for user libraries
Schema evolution risks user data loss, which is a direct release blocker.
PRD-021 • Critical
Autosave default semantics
Core journeys like rate-to-save and tag-to-save will behave unexpectedly without these defaults.
PRD-057 • Critical
Secret handling and server-only elevated keys
Credential leakage and invalid benchmark security posture become likely.
